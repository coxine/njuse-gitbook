# 人机交互系统

介绍软硬件产品的交互设计与评估。冯老师人特别好，给分也不错。

## 课程项目

4 人一组，分为两类：

* 开发类：开发一个能解决现有用户痛点，且交互性良好的产品。
* 原型类：选择具有明确应用场景的项目，进行基于高保真原型的交互设计（提交完整用户分析与设计文档：人物角色、原型迭代设计，交互评估实验等）

大部分团队都选择了开发类项目，此类项目评分的重点在于**解决用户的痛点**和**界面设计的美观**，应避免过多侧重于技术本身或选择比较大众的主题，大家在选题和开发时应多加注意。

{% hint style="info" %}
建议大家在选题前后多和冯老师沟通交流，听听冯老师的意见建议。
{% endhint %}

{% hint style="info" %}
比如站长的小组开发了一个“游迹”，用白板为载体，关联高德地图绘制旅行线路，记录旅行攻略。解决了目前旅游时用小红书做攻略在地理位置上不直观，需频繁切换地图应用和小红书的痛点，获得了课程项目的满分。😋

我们的 Github 仓库，欢迎大家参考：[前端](https://github.com/coxine/WiTrace-frontend) [后端](https://github.com/coxine/WiTrace-backend)
{% endhint %}

此外，项目还需录制场景视频，大家时间有余可以研究研究剪辑和运镜，这样会显得拍出来的视频很专业🤓

## 期末考试

最后一节课复习课会**公布去年的期末真题**，以便了解题型；还会复习一下重点内容，应做好记录。

选择题以 MOOC 真题为主。大题参考往年卷，最后几题考点一般都比较固定。
